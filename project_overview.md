# 学校用語辞書 (JP ⇔ PT) プロジェクト概要

## 1. プロジェクト概要

日本の学校に通う外国籍児童・生徒の保護者（主にポルトガル語話者）を対象とした、**学校用語の日英ポ（日本語・英語・ポルトガル語）Web辞書**を作成するプロジェクトです。
学校からの配布物や連絡帳などで頻出する用語を、誤解なく理解・伝達できるよう支援することを目的としています。

*   **開発フェーズ**: MVP (v0.1) - 初期100語の搭載
*   **ターゲット**: 保護者（スマホ利用想定）
*   **主な機能**: 用語検索、カテゴリ分類、「先生に見せる」拡大表示、お気に入り
*   **技術スタック**:
    *   フロントエンド: React (Vite / Next.js) + Static Export
    *   データ: JSONファイル (`terms.json`, `sources.json`)
    *   ホスティング: Vercel / GitHub Pages 等 (静的運用)

---

## 2. ファイル構成と詳細

プロジェクトの主要なファイルとその役割は以下の通りです。

### 📂 `HLD.md` (High Level Design / 基本設計書)
プロジェクトの全体像、設計思想、要件定義をまとめたドキュメントです。
*   **目的・背景**: なぜこのツールが必要か (保護者の負担軽減、学校とのコミュニケーション円滑化)。
*   **スコープ**: v0.1 (MVP) で実装する機能と、将来の拡張予定。
*   **データ方針**: **「既存PDFの転載はNG」「独自データとして再編集」** という重要なポリシーが定義されています。
*   **機能要件**: 検索ロジック、UI仕様 (用語カード、先生に見せるモードなど)。

### 📂 `sources.json` (出典管理データ)
用語データの元となる「出典 (Sources)」を管理するファイルです。
*   **目的**: 情報の信頼性を担保しつつ、著作権・利用条件を厳格に管理するため。
*   **主な項目**:
    *   `meta.policy`: 転載禁止ルール、利用条件URL保持のルール。
    *   `sources`: 各出典の配列。
        *   `id`: `S001`, `S002` などのユニークID。
        *   `license`: 利用条件 (`no_reupload`, `unclear` 等) とその根拠 (`evidence`)。
        *   `status`: 確認済み (`confirmed`) か要確認 (`needs_review`) か。
    *   `nextAddCandidates`: 今後追加を検討している出典候補リスト。

### 📂 `terms.json` (用語辞書データ)
アプリケーションで利用する実際の辞書データ本体です。
*   **目的**: フロントエンドが読み込んで検索・表示するためのマスターデータ。
*   **主な項目**:
    *   `categories`: 用語の分類定義 (`C01`: 提出・連絡, `C02`: 行事, `C03`: 場所, `C04`: 基本)。
    *   `terms`: 用語データの配列。
        *   `id`: `T0001` 等のユニークID。
        *   `ja`: 日本語表記。
        *   `pt`: ポルトガル語訳。
        *   `category_id`: 所属カテゴリ。
        *   `source_ids`: 出典ID (`sources.json` のIDと紐付け)。
        *   (v0.1では約100語が収録されています)
